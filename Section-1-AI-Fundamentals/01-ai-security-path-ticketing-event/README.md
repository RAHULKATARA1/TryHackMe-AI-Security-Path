## 🧠 Key Concepts

### Why AI Security?
AI is being rapidly integrated into production systems — from LLM-powered chatbots to automated ML pipelines. This creates a brand new attack surface that traditional security training doesn't cover. Attackers are already exploiting:
- Prompt injection vulnerabilities in LLM apps
- Poisoned training data in ML pipelines
- Insecure AI supply chains (malicious model weights, datasets)
- Data leakage through AI inference endpoints

### What This Path Covers
| Section | Focus |
|---------|-------|
| AI Fundamentals | How AI/ML systems work, their components, and basic threat landscape |
| Secure AI Systems | LLM security, threat modelling, and reconnaissance of AI infrastructure |
| Prompt Security | Prompt injection, jailbreaking, and building prompt defences |
| AI Supply Chain Security | Attacks on model sources, datasets, and dependencies |
| Data Poisoning | RAG security, training data manipulation, and information disclosure |
