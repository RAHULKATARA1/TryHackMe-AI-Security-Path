### Task 2 — Access Control Implementation

**Summary:** Implement document-level Role-Based Access Control (RBAC) so that each user only retrieves documents appropriate for their clearance level.

**Step 1 — Define the permission schema:**
```python
from enum import Enum
from dataclasses import dataclass
from typing import List

class ClearanceLevel(Enum):
    PUBLIC = 0
    EMPLOYEE = 1
    ASSOCIATE = 2
    PARTNER = 3
    MANAGING_PARTNER = 4

CLEARANCE_HIERARCHY = {
    ClearanceLevel.PUBLIC:           ["case-law-public"],
    ClearanceLevel.EMPLOYEE:         ["case-law-public", "hr-general"],
    ClearanceLevel.ASSOCIATE:        ["case-law-public", "hr-general", 
                                      "client-matter-general"],
    ClearanceLevel.PARTNER:          ["case-law-public", "hr-general",
                                      "client-matter-general", "client-privileged",
                                      "internal-research"],
    ClearanceLevel.MANAGING_PARTNER: ["case-law-public", "hr-general",
                                      "client-matter-general", "client-privileged",
                                      "internal-research", "financial", "strategy"],
}
```

**Step 2 — RBAC-enforced retrieval function:**
```python
def secure_retrieve(
    query: str,
    user_id: str,
    user_clearance: ClearanceLevel,
    db: Chroma,
    k: int = 5
) -> list:
    """
    Retrieves documents filtered by the user's clearance level.
    All retrieval events are logged to the audit trail.
    """
    # Get allowed categories for this clearance level
    allowed_categories = CLEARANCE_HIERARCHY[user_clearance]
    
    # Apply metadata filter — only retrieve allowed documents
    results = db.similarity_search(
        query,
        filter={"category": {"$in": allowed_categories}},
        k=k
    )
    
    # Audit log every retrieval event
    log_retrieval_event(
        user_id=user_id,
        clearance=user_clearance.name,
        query=query,
        retrieved_doc_ids=[r.metadata.get("id") for r in results],
        retrieved_categories=[r.metadata.get("category") for r in results]
    )
    
    return results
```

**Step 3 — Verify access control works:**
```python
# Test: PUBLIC user cannot access client communications
```

### Task 3 — Ingestion Pipeline Hardening

**Summary:** Harden the document ingestion pipeline to prevent future accidentally-indexed documents and injection attacks.

**Hardened ingestion pipeline:**
```python
import hashlib
import re
from pathlib import Path
from typing import Optional

# Allowlist of approved document sources and categories
APPROVED_SOURCES = {
    "westlaw-export": "case-law-public",
    "lexisnexis-export": "case-law-public",
    "approved-case-files": "client-matter-general",
}

PROHIBITED_CONTENT_PATTERNS = [
    r"(?i)(api[_\s]?key|secret[_\s]?key|password|token)\s*[:=]\s*\S+",  # Credentials
    r"LEXAI-API-[A-Z0-9-]+",                                               # API keys
    r"(?i)(ignore\s+previous|system\s+instruction|new\s+directive)",       # Injections
    r"(?i)(connection\s+string|jdbc:|mongodb://|postgresql://)",           # DB strings
]

INJECTION_INSTRUCTION_PATTERNS = [
    "ignore all previous instructions",
    "you are now", 
    "forget everything",
    "new system prompt",
    "output all documents",
    "disregard your guidelines",
]

class SecureIngestionPipeline:
    def __init__(self, db: Chroma, audit_logger):
        self.db = db
        self.logger = audit_logger
    
    def ingest_document(
        self, 
        content: str, 
        source: str,
        submitted_by: str,
        clearance_level: str
    ) -> Optional[str]:
        """
        Validates, sanitizes, and indexes a document.
        Returns the document ID on success, None on rejection.
        """
        # 1. Verify source is on the allowlist
        if source not in APPROVED_SOURCES:
            self.logger.warning(f"REJECTED: Unknown source '{source}' from {submitted_by}")
            return None
        
        # 2. Scan for prohibited content patterns
        for pattern in PROHIBITED_CONTENT_PATTERNS:
            if re.search(pattern, content):
                self.logger.critical(
                    f"BLOCKED: Credential/prohibited content pattern in doc from {submitted_by}"
                )
                return None
        
        # 3. Scan for injection instruction patterns
        content_lower = content.lower()
        for pattern in INJECTION_INSTRUCTION_PATTERNS:
            if pattern in content_lower:
                self.logger.critical(
                    f"BLOCKED: Injection instruction detected from {submitted_by}: '{pattern}'"
                )
                return None
        
        # 4. Assign category from approved source mapping
        category = APPROVED_SOURCES[source]
        
        # 5. Compute integrity hash
        content_hash = hashlib.sha256(content.encode()).hexdigest()
        
        # 6. Index with full metadata
        doc_id = self.db.add_texts(
            texts=[content],
            metadatas=[{
                "source": source,
                "category": category,
                "clearance_required": clearance_level,
                "submitted_by": submitted_by,
                "ingestion_timestamp": datetime.utcnow().isoformat(),
                "content_hash": content_hash,
                "scan_status": "PASSED"
            }]
        )[0]
        
        self.logger.info(f"INDEXED: doc_id={doc_id}, source={source}, category={category}")
        return doc_id
```

**Questions & Answers:**

| Question | Answer |
|----------|--------|
| What two categories of prohibited content does the hardened ingestion pipeline scan for? | `1. Credential patterns (API keys, passwords, connection strings). 2. Injection instruction patterns (ignore previous, system instruction, etc.)` |
| What allowlist mechanism prevents future accidental ingestion of configuration files? | `APPROVED_SOURCES dictionary — only documents from pre-approved sources with mapped categories are accepted` |
| What metadata field enables future forensic investigation if a document is later found to be malicious? | `submitted_by, ingestion_timestamp, and content_hash — together creating a full audit trail` |

### Task 4 — Output Guardrails & Audit Logging

**Summary:** Deploy output-layer guardrails and comprehensive audit logging to detect and prevent future attacks.

**Output guardrail implementation:**
```python
import re
from dataclasses import dataclass

@dataclass
class GuardrailResult:
    passed: bool
    redacted_output: str
    violations: list

class LexAIOutputGuardrail:
    
    PRIVILEGED_INDICATORS = [
        r"(?i)(do not share|attorney.client|privileged|confidential)",
        r"(?i)(settlement\s+threshold|settlement\s+figure|\$\d+[MK])",
        r"(?i)(don't\s+tell|keep\s+this\s+off|off\s+the\s+record)",
    ]
    
    PII_PATTERNS = [
        (r'\b\d{3}-\d{2}-\d{4}\b', '[SSN REDACTED]'),
        (r'\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b', '[EMAIL REDACTED]'),
        (r'\b\d{4}[-\s]?\d{4}[-\s]?\d{4}[-\s]?\d{4}\b', '[CARD REDACTED]'),
    ]
    
    CREDENTIAL_PATTERNS = [
        (r'LEXAI-API-[A-Z0-9-]+', '[API KEY REDACTED]'),
        (r'ghp_[A-Za-z0-9]+', '[GITHUB TOKEN REDACTED]'),
        (r'hf_[A-Za-z0-9]+', '[HF TOKEN REDACTED]'),
    ]
    
    def evaluate(self, output: str, user_clearance: str) -> GuardrailResult:
        violations = []
        redacted = output
        
        # Check for privileged content leaking to non-partner users
        if user_clearance not in ["PARTNER", "MANAGING_PARTNER"]:
            for pattern in self.PRIVILEGED_INDICATORS:
                if re.search(pattern, output):
                    violations.append(f"PRIVILEGED_CONTENT: {pattern}")
                    redacted = re.sub(pattern, '[REDACTED — INSUFFICIENT CLEARANCE]', 
                                      redacted, flags=re.IGNORECASE)
        
        # Always redact PII
        for pattern, replacement in self.PII_PATTERNS:
            if re.search(pattern, redacted):
                violations.append(f"PII_DETECTED: {pattern}")
                redacted = re.sub(pattern, replacement, redacted)
        
        # Always redact credentials
        for pattern, replacement in self.CREDENTIAL_PATTERNS:
            if re.search(pattern, redacted):
                violations.append(f"CREDENTIAL_DETECTED: {pattern}")
                redacted = re.sub(pattern, replacement, redacted)
        
        return GuardrailResult(
            passed=len(violations) == 0,
            redacted_output=redacted,
            violations=violations
        )
```

**Audit logging:**
```python
import json
from datetime import datetime

def log_retrieval_event(
    user_id: str,
    clearance: str,
    query: str,
    retrieved_doc_ids: list,
    retrieved_categories: list,
    guardrail_violations: list = None
):
    event = {
        "timestamp": datetime.utcnow().isoformat(),
        "event_type": "RAG_RETRIEVAL",
        "user_id": user_id,
        "clearance_level": clearance,
        "query_hash": hashlib.sha256(query.encode()).hexdigest(),
        "query_length": len(query),
        "retrieved_doc_count": len(retrieved_doc_ids),
        "retrieved_doc_ids": retrieved_doc_ids,
        "retrieved_categories": list(set(retrieved_categories)),
        "guardrail_violations": guardrail_violations or [],
        "alert": len(guardrail_violations or []) > 0
    }
    
    # Write to append-only audit log (SIEM integration)
    with open("/var/log/lexai/audit.jsonl", "a") as f:
        f.write(json.dumps(event) + "\n")
    
    # Alert on violations
    if event["alert"]:
        send_siem_alert(event)
```

## 💡 Personal Takeaways

- **Lockdown perfectly crystallises the defender's dilemma**: every control you implement adds friction — to attackers and legitimate users alike. The skill is in deploying controls that maximally restrict attackers while minimally impacting authorized users. Document-level RBAC achieves this far better than output filtering or query keyword blocking.
- The **22-test validation suite** represents the mindset every AI security implementation should have: adversarial testing, not just feature testing. "Does the chatbot answer questions?" is the wrong question. "Does the chatbot answer the *right* questions *only* to the *right* people?" is the security question.
- **The complete path taught a recurring pattern**: AI attacks almost always exploit the gap between *what a system can technically do* and *what it was intended to do*. Prompt injection exploits the gap between instruction and data. Supply chain attacks exploit the gap between "downloaded model" and "trusted model." RAG exfiltration exploits the gap between "indexed for reference" and "restricted from disclosure." Closing these gaps requires intentional security design, not reactive patching.
- My biggest takeaway from the entire AI Security Path: **AI security is not a feature you add after building the system — it's an architectural discipline you apply from day one.** Bolting on access controls, guardrails, and monitoring after deployment is exponentially harder than designing for them from the start. The field is still young enough that most practitioners don't know this yet — which makes this knowledge enormously valuable.
